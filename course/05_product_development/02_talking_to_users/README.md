# Module: Talking to Users

YCombinator says you should only do two things: "write code [and] talk to users." But how do you do it right?

## Submodules

1. [Intellectual Honesty in User Conversations](01_intellectual_honesty_in_user_conversations.md) - Stop trying to convince users; listen to what they're really saying
2. [Jobs to be Done](02_jobs_to_be_done.md) - What job did you hire that product to do?
3. [Measuring Customer Satisfaction (CSAT)](03_measuring_customer_satisfaction_csat.md) - How to quantify and track customer satisfaction
4. [Reading Support Emails](04_reading_support_emails.md) - Why everyone should read customer support emails
5. [The Limits of Being Data-Driven](05_the_limits_of_being_data_driven.md) - Why data-driven decisions are hard (sometimes impossible)
6. [Backlog Size and Customer Conversations](07_backlog_size_and_customer_conversations.md) - The size of your backlog is inversely proportional to how often you talk to customers
